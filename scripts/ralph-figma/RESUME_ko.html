Figma 이식 루프 — 재개 가이드

네트워크 끊김 / 세션 종료 후에도 완료 지점부터 이어서 진행할 수 있게 하는 문서. 상태는 이미 디스크에 durable하게 저장됨.

상태 저장 위치 (이미 durable)

  • scripts/ralph-figma/prd.json — 각 스토리 passes(완료)와 notes(생성 nodeId). 네트워크 끊겨도 유지.
  • scripts/ralph-figma/progress.txt — ## Codebase Patterns에 nodeId 맵·학습.
  • 매 스토리가 멱등(이름/nodeId로 기존 프레임 찾아 이어그림) → 중간에 죽어도 다시 돌리면 완료 지점부터, 중복 없이 이어감.

재개 방법 (원클릭)

Figma 플러그인은 재접속할 때마다 채널이 바뀐다(예: 5evd9cs8 → 6rkd10x7). 재개 시 보통 새 채널을 넘긴다.
cd /Users/osein/cook-assistance-wireframe

# 1) 플러그인 열고 Connect → 표시된 채널 확인
# 2) 그 채널로 재개 (브리지 자동 기동 + 중복 정리 + 감시견 가동)
./scripts/ralph-figma/resume.sh <새_채널>

# 채널 그대로면:
./scripts/ralph-figma/resume.sh

진행 확인

jq -r '"완료 "+([.userStories[]|select(.passes==true)]|length|tostring)+"/106, 다음="+([.userStories[]|select(.passes==false)]|sort_by(.priority)|.[0].id)' scripts/ralph-figma/prd.json
tail -f /tmp/figma-watchdog.log

감시견 동작

  • 1분 주기 싼 probe로 플러그인 생사 확인 → 살아있으면 supervise 배치로 완료까지, 죽어 있으면 대기(토큰 낭비 없음).
  • 네트워크 끊기면 DEAD 취급하고 대기 → 복구되면 자동 재개.
  • 주의: nohup이라 터미널 닫아도 살지만 재부팅/로그아웃 시 죽음 → resume.sh로 재기동.

트러블슈팅

  • 계속 DEAD: 플러그인 창 닫힘/채널 불일치 → Connect 후 채널 재확인.
  • 브리지 죽음: cd /Users/osein/cursor-talk-to-figma-mcp && bun run src/socket.ts(resume.sh 자동 처리).
  • 중복 그리기 우려: 감시견은 1개만. pkill -f ralph-figma/watchdog.sh 후 resume.sh.
