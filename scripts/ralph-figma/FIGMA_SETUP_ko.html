Figma 쓰기형 MCP 셋업 가이드 사용자 직접

와이어프레임을 Figma에 그리려면 쓰기 가능한 MCP가 필요합니다. Framelink·REST·Dev Mode MCP는 모두 읽기 전용이라 프레임/텍스트 생성 불가. 그리기가 되는 옵션은 플러그인 브리지형(예: cursor-talk-to-figma-mcp, "TalkToFigma") — MCP 서버 + Figma 플러그인이 로컬 WebSocket으로 통신하며 create_frame/create_text/export_node_as_image 등을 노출합니다.

1~4단계는 사람만 할 수 있습니다(플러그인 설치·파일 열기·브리지 실행·채널 조인). 완료 후 세션 재시작하면 에이전트가 F0 스파이크부터 자율로 그립니다.

0. 사전 요구

  • Figma 데스크톱/편집 권한 + 그릴 대상 파일.
  • bun(또는 node/npx). (brew install oven-sh/bun/bun · https://bun.sh)

1. WebSocket 브리지(릴레이) 실행

bunx cursor-talk-to-figma-socket     # 또는 저장소 클론 후 `bun socket`

정확한 패키지명·명령은 설치 시점 문서로 확인(카테고리=플러그인 브리지 쓰기 MCP는 안정적).

2. Figma 플러그인 설치 + 대상 파일 열기

⚠️ 이 플러그인은 커뮤니티 검색에 안 뜰 수 있습니다. 아래 A(직접 링크) 또는 B(수동 임포트)로 설치.

A. 커뮤니티 직접 링크 — 검색창 대신 URL로 진입:
Talk To Figma MCP Plugin

B. 로컬 개발 플러그인 수동 임포트 (가장 확실, Figma 데스크톱 앱 전용):

git clone https://github.com/grab/cursor-talk-to-figma-mcp.git
# 원 저장소 sonnylazuardi/... 가 grab/ 로 이전됨
  • Figma 데스크톱 → 메뉴 → Plugins → Development → Import plugin from manifest…
  • 클론 폴더의 src/cursor_mcp_plugin/manifest.json 선택 → 개발 플러그인 등록.
  • 대안: Claude 특화 포크 arinspunk/claude-talk-to-figma-mcp.
  • 대상 파일을 열고 페이지를 Cookflow Wireframes로 준비.

3. 플러그인 실행 → 채널 조인

  • 플러그인 → "Connect" → 표시되는 채널 ID 복사.
cp .env.example .env
# .env 의 FIGMA_CHANNEL 를 채널 ID로 교체

scripts/ralph-figma/prd.json의 figma.channel도 같은 값으로 교체.

4. MCP 등록 확인

리포 루트 .mcp.json에 TalkToFigma 서버가 정의돼 있습니다. 필요 시 명령/패키지명을 실제 설치본에 맞게 수정.

5. 세션 재시작 (중요)

MCP 툴은 세션 시작 시 바인딩. .mcp.json 추가 후 Claude Code 세션 재시작 필요.

./scripts/ralph-figma/ralph.sh --tool claude 1     # 1회 스파이크
./scripts/ralph-figma/supervise.sh 3 3             # 통과 시 자율 루프

트러블슈팅

  • 툴이 안 보임: 세션 재시작 안 됨 / .mcp.json 명령 오류 / bun 미설치.
  • join_channel 실패: 브리지(1) 미실행 / 플러그인(3) 미연결 / 채널 ID 불일치.
  • export 실패: 플러그인·파일 편집 권한 확인.
  • 헤드리스 연결 불안정: stateful WebSocket이라 --tool claude 1로 1회 검증 후 supervise 사용.
