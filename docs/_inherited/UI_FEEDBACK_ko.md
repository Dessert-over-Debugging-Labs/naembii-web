# Cookflow Android — UI 가이드 / 피드백 종합 (필독)

> 이 문서는 **빌드 전·중 반드시 읽고 따르는 UI 바이블**이다. 이전 라운드에서 받은 모든 UI 피드백 + v3_2 디자인 의도 + "모든 버튼 실제 동작" 원칙을 모았다.
> 디자인 레퍼런스(진실): `docs/reference/cookflow_v3_2_interactive.html` (화면 안 버튼까지 클릭 동작). 색/레이아웃/간격을 최대한 일치.
> 단위: Compose 기준 `sp`(폰트)·`dp`(간격/크기).

## 0. 최우선 원칙
1. **모든 버튼·카드·칩이 실제로 동작한다.** 화면 안의 어떤 탭 가능한 요소도 "장식"이면 안 된다(dead UI 금지). 누르면 화면 전환/선택/상태변경 + **press 피드백**(scale·ripple)이 있어야 한다.
2. **모바일 네이티브 느낌** — 웹 대시보드를 욱여넣은 느낌 금지. "만지고 싶은" 컴포넌트.
3. **전면 다크 테마 고정**, v3_2 색 일치.

## 1. 다크 디자인 토큰 (v3_2 정확값)
| 역할 | 값 |
|---|---|
| 배경 bg | `#0A0A0A` |
| 서피스 surface | `#161616` |
| 테두리 border | `#242424` |
| 포인트 orange | `#FF6B35` (dim `rgba(255,107,53,.12)`, border `rgba(255,107,53,.25)`) |
| 텍스트 text | `#F0EBE3` |
| 서브 sub | `#999999` / muted `#555555` |
| 그린 | `#4ADE80` · 블루 `#60A5FA` · 옐로우 `#FBBF24` |

폰트: 시스템(Android Roboto / iOS는 별도 담당). 라운드: 카드 16dp 안팎, pill 999.

## 2. 타이포그래피 (이전 피드백 핵심 — 반드시 반영)
- **본문 14~16sp, 서브 12~13sp.** 과대 헤딩 금지(조리 안내문도 16sp선, 헤딩 22~24sp 이하로 절제).
- **행간(line-height) 1.5~1.6** 으로 넓혀 가독성 확보.
- **위계는 크기 남발이 아니라 weight(Bold 700 / Medium 600 / Regular 400)로** 분리.
- **한글 줄바꿈**: 단어 중간에서 어색하게 끊기지 않게(Compose: `softWrap` + 적절한 width, 필요 시 `LineBreak.Heading`/`break-word` 대응). 좌우 마진 확보.

## 3. 레이아웃 · 여백 (8dp 그리드)
- 모든 margin/padding/gap을 **8의 배수(8/16/24/32dp)** 로 통일.
- 카드 radius·padding 일관(투박함 제거).
- **정보 밀도 완화**: 바둑판/촘촘한 그리드·체크리스트 지양 → 요소 간 간격 과감히 넓혀 시선 분산↓.
- **스크롤 유도**: 한 화면에 다 욱여넣지 말고 하단 요소가 자연스럽게 잘려 스크롤을 인지시키기.

## 4. 컴포넌트 표준
- **터치 타깃 최소 48dp**(버튼/칩/스테퍼/탭 포함).
- **인풋/버튼 높이·내부 텍스트 밸런스**: 버튼 글자 과대로 답답하지 않게(버튼 텍스트 15~16sp, 높이 ~48dp).
- **고정 하단 탭바**: 화면 하단 고정·균등 분할, 아이콘+텍스트 안 잘림, 활성 표시 일관, safe-area(시스템 바) 패딩 고려.

## 5. 핸즈프리 조리(Screen 4) 특수 UX — v3_2 의도
- **영상 우선**: 상단 ~32% 실제 유튜브 임베드(현재 단계 구간 자동 재생). 오버레이: STEP n/6 · 타임스탬프 · 재생바 · 재생/정지.
- **영상 동적 표시/숨김**: 단계 데이터 `showVideo:false`면 영상 슬라이드 업으로 숨고 비서+타이머가 중앙 확장. 탭/"영상 보기"로 다시 내림.
- **플로팅 AI 비서**: 작은 버블(마이크+파동) ↔ 위로 슬라이드 확장 대화 카드. 단계 진입/타이머 10초 전/정체 시 **선제 안내**. 이전 답변 말풍선 유지. 아래 스와이프로 최소화.
- **볼륨 더킹**: 비서 말하는 동안 영상 볼륨↓ + 반복, 끝나면 복구(네이티브는 AudioManager 오디오 포커스/TTS로 처리).
- **타이머**: 단계 카드 우상단 뱃지 → 탭 시 바텀시트(SVG/Canvas 링 + 프리셋 30초/1분/2분/5분 + 시작/정지/초기화).
- **음성명령 UI**(STT는 MVP 제외): 다음/이전/다시/타이머/영상/줄여줘.
- **페이싱 유지**: 자동 진행 금지, 사용자가 "다음"(음성/탭)으로 진행.

## 6. 페르소나 톤 (이모 / 엄마 / 셰프)
- 이모=따뜻·친근, 엄마=꼼꼼·챙김, 셰프=전문·간결. 같은 상황에서 멘트 톤이 실제로 다르게. 기능은 동일.

## 7. 접근성
- 대비 충분(다크 위 텍스트 AA 지향), 큰 터치, 핵심 컨트롤 contentDescription, 동적 상태(타이머/음성) 안내.

## 8. 화면별 클릭 동작 체크 (실앱 터치 — 전부 동작해야)
- 홈: 추천/트렌딩 카드·퀵칩·AI버블 탭 → 진입/선택
- 가져오기: 페르소나 선택, CTA→리뷰, 뒤로
- 리뷰: 재료/단계 탭(구간 seek 표시), CTA→조리, 뒤로
- 조리: 이전/다음 단계, 다시보기/반복, 타이머/음성 모달, 영상 토글, 비서 버블
- 완료: 별점, 평가(또만들래요/수정후에/별로예요), 사진 메모, 공유

## 9. 완료 화면(소셜) — v3_2
완성 사진+메모 → 레시피 저장(즉시) → 별점/평가 → **타인 후기 카드(닉네임·별점·태그)** → 친구 공유.
