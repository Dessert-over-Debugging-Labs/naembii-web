# Naembi 사용자 문구 톤 기준

## 기본 원칙

- 기능 구현 방식보다 사용자가 지금 할 수 있는 행동을 먼저 말한다.
- 안내는 `~해보세요`, 응답은 `~할게요`, 상태는 짧은 명사형으로 구분한다.
- 기술 용어는 권한·브라우저 오류 해결에 꼭 필요한 경우에만 사용하고 바로 다음 행동을 함께 적는다.
- 지원하지 않는 기능은 입력창처럼 보이게 두지 않고 지원 범위를 직접 알린다.
- 후기에는 완주 후 가벼운 감상을, 조리 팁에는 맛 조절·재료 대체처럼 다시 쓸 수 있는 정보를 안내한다.

## 음성비서 기준 문구

- 대기: `직접 말로 물어보거나, 추천 질문을 선택해보세요!`
- 권한 요청: `마이크 권한을 허용하면 바로 들을게요.`
- 듣는 중: `질문을 편하게 말해보세요. 말이 끝나면 답변할게요.`
- 입력 제한: `현재는 음성 또는 추천 질문만 지원해요`
- 상태 순서: `마이크 OFF → 마이크 ON · 듣는 중 → 인식 완료 → 응답 중 → 응답 완료`

## 자동 검증

```bash
npm run verify:copy-tone
```

기본 검증은 이전 문구, 구현 설명, 내부 개발 용어, 필수 문구 누락을 실패 처리한다. 명령조·기술 안내·기획자 중심 문구는 검토 경고로 출력한다.

경고까지 실패로 처리하려면 다음 명령을 사용한다.

```bash
npm run verify:copy-tone -- --strict
```
