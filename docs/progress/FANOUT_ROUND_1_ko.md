# Fan-out 검증 라운드 1 — cook 와이어프레임

작성: 2026-06-29

## 방식

- 병렬 렌즈 4개: 다크 디자인/UI_FEEDBACK, 모든 버튼 동작, cook3 핸즈프리·영상 정직성, 한국어·접근성.
- 대상: `app.html`, `docs/_inherited/*`, 대표 스크린샷.
- 판정 기준: 구체 코드 위치와 사용자 영향이 있는 항목만 확정. 문서 상속 자료 자체의 Android 과거 판정은 현재 웹 프로토타입 결함으로 보지 않는다.

## 확정 발견과 조치

| 심각도 | 발견 | 조치 |
|---|---|---|
| 심각 | cook3 vpanel 데모가 예시 발화 뒤 실제 단계를 자동 전진해 사용자 페이싱을 위반했다. | `runVp()`에서 단계 전진/타이머 자동 시작을 제거하고, 예시 답변/시뮬레이션 문구로 낮췄다. 실제 전진은 하단 `다음` 버튼으로만 수행한다. |
| 높음 | 수동 다음 이동 토스트가 `"다음 단계" 인식됨`이라 실제 음성 인식처럼 보였다. | 수동 이동 문구를 `다음 단계로 이동했어요`로 변경했다. |
| 높음 | 사이드바/해시 이동 시 재료·타이머·후기 시트가 남을 수 있었다. | `nav()`와 `applyHash()` 공통 정리에 `closeIngredients`, `closeTimer`, `closeReviewSheet`를 포함했다. |
| 높음 | 구버전 `cook`/`cook2` 아이콘 버튼과 영상 영역의 aria/정직 안내가 부족했다. | 두 화면의 뒤로/가이드/영상 영역에 role, tabindex, aria-label, 영상 폴백 안내를 추가했다. |
| 높음 | 상세/홈 카드가 여러 영상처럼 보이나 상세는 순두부 고정이었다. | 카드 클릭 시 비대표 영상은 “이 프로토타입은 순두부 상세만 연결” 토스트를 먼저 보여준다. |
| 높음 | cook3 첫 단계의 이전 버튼이 무반응이었다. | 첫 단계에서는 `첫 단계예요` 토스트를 제공한다. |
| 높음 | 핸즈프리 버튼이 켜져도 보이는 텍스트/아이콘이 `꺼짐` 상태로 남았다. | `aria-pressed`, 아이콘, 텍스트를 켜짐/꺼짐 상태에 맞춰 갱신한다. |
| 높음 | 링크 등록 필드가 입력처럼 보이지만 실제 입력이 아니었다. | 실제 URL input으로 교체하고 빈 값 검증을 추가했다. |
| 높음 | 48px 터치 기준 미달: 상세 탭, 닫기, 타이머 프리셋, 음성 가이드 닫기. | 각 컨트롤에 `min-height`/48px 터치 영역을 적용했다. |
| 높음 | 완료 화면이 저장/별점/메모/공유/후기 흐름 없이 단순했다. | 완료 화면에 저장 완료, 별점 남기기, 사진 메모, 친구 공유, 타인 팁 카드를 추가했다. |
| 중간 | 별점 선택과 별점 표시의 보조기술 상태가 부족했다. | 별점 입력을 radiogroup/radio 구조로 바꾸고 표시 별은 aria label/hidden을 보강했다. |
| 중간 | 조리 단계 카드가 클릭 가능하지만 키보드 접근이 부족했다. | `.scard`에 role/button/tabindex/aria-label을 부여하고 공통 키보드 활성화에 포함했다. |
| 중간 | 영어/혼용 카피: `cook`, `A/B/C 단계`, `YouTube` 등. | 브랜드를 `쿡`, 단계 표시를 `1단계` 형식, 사용자 노출 유튜브 표기를 한국어로 정리했다. |

## 보류/기각

- 조리 영상 높이 32% 요구: 200px에서 260px로 확장해 반영했다.
- v3_2 외 직접 색상 일부: `--surface2` 같은 보조 토큰은 유지하되 주요 표면/보더/텍스트/그린은 토큰으로 정리했다.
- 실제 유튜브 재생 검증: 이 HTML 프로토타입은 정적 미리보기만 제공하므로 실기기 재생 검증 대상이 아니다. 화면에는 “정적 미리보기 · 실제 재생은 실기기에서 확인”으로 표기했다.

## 재검증

- `<script>` 추출 `node --check` 통과.
- Chrome headless 스크린샷 생성: `W3-1-cook3-fixed.png`, `W3-1-complete-fixed.png`, `W3-1-sheet-fixed.png`.
- 사용자 노출 카피 스캔: `재생 중`, `STEP`, `Mic`, `YouTube`, `쉐프`, 장식 이모지 누수 없음. 남은 `mic-off`/`off`는 아이콘명·CSS 클래스명이다.
- Chrome 자체 GPU/프로필 로그는 앱 JS 오류가 아닌 실행환경 노이즈로 분리했다.

## 결론

라운드1에서 확정된 결함은 모두 즉시 보정했다. 추가 FIX 스토리 주입 없이 `W3-2` 통합 E2E와 최종 리포트로 진행한다.
