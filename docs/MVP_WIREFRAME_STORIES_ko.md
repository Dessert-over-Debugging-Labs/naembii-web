# MVP 와이어프레임 수정 스토리라인

작성일: 2026-07-01

## 목적

최근 대화에서 확정된 와이어프레임 수정 요구를 별도 문서로 고정한다. 이후 Ralph loop는 이 문서와 `scripts/ralph/prd.json`의 `MVP-*` 스토리를 대조해 누락 여부를 검증한다.

## 확정 요구

1. 모든 버튼은 하나씩 클릭 검증한다. 화면이 있는 MVP 기능은 실제 화면 또는 시트로 이동해야 하며, MVP 밖 기능은 다음 버전 안내 토스트를 보여야 한다.
2. 홈의 `이미 만든 요리`는 저장과 별개다. “0회 터치” 같은 의미 약한 지표 대신 왜 만든 음식인지 알 수 있는 기록 근거를 보여준다.
3. `요리 시작하기` 플로팅 버튼은 좋지만, 버튼 아래 하단 직사각형 영역이 배경을 가리면 안 된다.
4. 조리 모드에는 마지막 `요리 완성` 버튼이 필요하다. 이전/다음 버튼은 제거하고, 단계 이동은 가운데 단계 카드 스와이프/휠/키보드로 처리한다.
5. 완료 화면은 완성 상태를 먼저 보여준다. 저장/별점/팁/공유는 이후 액션으로 분리한다.
6. `다른 사용자의 팁`은 조리 전 상세에서도 볼 수 있어야 한다.
7. `다른 사용자를 위한 팁 남기기` 화면이 있어야 한다.
8. 실제 소스 테스트 케이스를 분리한다: YouTube 긴 영상/짧은 영상, Instagram 영상, 전문 요리사/일반 사용자. 앱 내부 영상 처리에서는 실제 재생을 단정하지 않고 정적 미리보기와 분석 상태를 정직하게 보여준다.
9. 후속 할 일로 Figma MCP 기반 와이어프레임 제작과 기술명세서(Notion 또는 Excel) 제작을 등록한다. 지금은 실행하지 않고 백로그로만 둔다.
10. 홈에는 최근 만든 요리가 꼭 보일 필요가 없다. 홈은 추천/검색/영상 등록 중심으로 두고, 요리북은 별도 세부 페이지에서 최근 조리, 저장한 레시피, 기록 기반 추천, 기록 메모를 보여준다.

## Ralph 스토리 분해

### MVP-01 요구사항 문서화·PRD 등록
- 이 문서를 생성한다.
- `scripts/ralph/prd.json`에 `MVP-*` 스토리를 `passes:false`로 등록한다.
- 검증: 문서와 PRD가 같은 요구를 담고, `validate-prd.sh`가 통과한다.

### MVP-02 상세 CTA 투명 플로팅 보정
- 상세 화면의 `요리 시작하기`는 플로팅 버튼만 보이고, 하단 직사각형 배경이 콘텐츠를 가리지 않는다.
- 검증: `#detail` 스크린샷에서 CTA 주변 배경이 보이고 버튼 클릭 시 `#cook3`로 이동한다.

### MVP-03 조리 모드 종료·스와이프 페이싱
- cook3 하단에서 이전/다음 버튼을 제거한다.
- `요리 완성` 버튼을 추가한다.
- 단계 카드는 스와이프/휠/키보드로 이동 가능하고 힌트를 보여준다.
- 검증: 완료 버튼 클릭 시 `#complete`, 스와이프 시 단계 번호 변경.

### MVP-04 완료·팁·후기 액션 분리
- 완료 화면은 완성 결과를 먼저 보여준다.
- `팁 남기기`는 별도 화면으로 이동한다.
- `후기 남기기`는 기존 후기 작성 시트로 이동한다.
- MVP 밖 기능은 다음 버전 안내만 한다.
- 검증: 각 버튼이 하나의 액션만 수행한다.

### MVP-05 만든 요리 기록 카피 정합
- `최근 만든 요리`의 근거는 완료 일시, 메모/팁 여부, 다시 만들기 같은 재방문 동기로 표현한다.
- `성공`과 `다시 만들기`처럼 서로 다른 차원의 상태를 나란히 두지 않는다.
- 검증: `0회 터치`, 의미 불명 상태 문구가 남지 않는다.

### MVP-06 버튼별 클릭 매트릭스 검증
- 홈, 상세, 조리, 완료, 팁 화면의 주요 버튼을 CDP로 하나씩 클릭한다.
- 각 버튼은 화면 전환, 시트 열림, 토스트 중 하나를 만든다.
- 검증: 콘솔 에러 0, dead UI 0.

### MVP-07 외부 영상 소스 테스트 케이스 정리
- YouTube long/short, Instagram, 전문 요리사/일반 사용자 케이스를 샘플 데이터로 분리한다.
- 등록 시트에서 케이스를 선택해 내부 분석 상태가 바뀌도록 한다.
- 검증: 각 케이스가 플랫폼/제작자 유형/영상 길이/정직한 미리보기 상태를 보여준다.

### MVP-08 Figma·기술명세서 백로그 등록
- Figma MCP로 와이어프레임 제작.
- Notion 또는 Excel에 기술명세서 제작.
- 지금은 실행하지 않고 후속 백로그로만 둔다.
- 검증: TODO 문서 또는 PRD 백로그에 남는다.

### MVP-09 요리북 세부 페이지 분리
- 홈에서 최근 만든 요리 섹션을 제거한다.
- 하단 `요리북` 탭은 별도 `cookbook` 페이지로 이동한다.
- 요리북에는 최근 조리, 저장한 레시피, 기록 기반 추천, 기록 메모를 보여준다.
- 검증: 홈 스크린샷에 최근 만든 요리가 없고, `#cookbook` 스크린샷에 기록/저장/추천이 보인다.
