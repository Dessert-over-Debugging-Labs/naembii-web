# MVP 와이어프레임 수정 스토리라인

작성일: 2026-07-01

## 목적

최근 대화에서 확정된 와이어프레임 수정 요구를 별도 문서로 고정한다. 이후 Ralph loop는 이 문서와 `scripts/ralph/prd.json`의 `MVP-*` 스토리를 대조해 누락 여부를 검증한다.

## 확정 요구

1. 모든 버튼은 하나씩 클릭 검증한다. 화면이 있는 MVP 기능은 실제 화면 또는 시트로 이동해야 하며, MVP 밖 기능은 다음 버전 안내 토스트를 보여야 한다.
2. 홈의 `이미 만든 요리`는 저장과 별개다. “0회 터치” 같은 의미 약한 지표 대신 왜 만든 음식인지 알 수 있는 기록 근거를 보여준다.
3. `요리 시작하기` 플로팅 버튼은 좋지만, 버튼 아래 하단 직사각형 영역이 배경을 가리면 안 된다.
4. 조리 모드에는 마지막 `요리 완성` 버튼이 필요하다. 이전/다음 버튼은 제거하고, 단계 이동은 가운데 단계 카드 스와이프/휠/키보드로 처리한다.
5. 완료 화면은 완성 상태를 먼저 보여준다. 저장/별점/팁/공유는 이후 액션으로 분리한다.
6. `다른 사용자의 팁`은 조리 전 상세에서도 볼 수 있어야 한다.
7. `다른 사용자를 위한 팁 남기기` 화면이 있어야 한다.
8. 실제 소스 테스트 케이스를 분리한다: YouTube 긴 영상/짧은 영상, Instagram 영상, 전문 요리사/일반 사용자. 앱 내부 영상 처리에서는 실제 재생을 단정하지 않고 정적 미리보기와 분석 상태를 정직하게 보여준다.
9. 후속 할 일로 Figma MCP 기반 와이어프레임 제작과 기술명세서(Notion 또는 Excel) 제작을 등록한다. 지금은 실행하지 않고 백로그로만 둔다.
10. 홈에는 최근 만든 요리가 꼭 보일 필요가 없다. 홈은 추천/검색/영상 등록 중심으로 두고, 요리북은 별도 세부 페이지에서 최근 조리, 저장한 레시피, 기록 기반 추천, 기록 메모를 보여준다.
11. 조리 과정 화면만 블랙 배경을 유지한다. 블랙+오렌지 조합은 주력안에서 제외하고, 화이트 기본안과 허브/스틸/크림 보조안을 비교 가능하게 둔다.
12. 조리 화면은 다크 모드로 유지하고, 홈/상세/요리북/완료 등 일반 앱 화면은 반대 톤인 라이트 모드로 전환한다. 하단 내비게이션은 `홈 / 등록 / 요리북` 3개만 둔다. 설정은 하단 탭에서 제거하고 홈 상단 계정 버튼으로 진입한다.
13. 레시피 등록은 사용자가 영상 링크만 붙여넣으면 된다. YouTube/Shorts/Instagram, 전문 요리사/일반 사용자 같은 출처·형식은 앱이 자동 감지해야 하며 사용자가 별도로 고르거나 입력하지 않는다.
14. 라이트 화면이 눈에 아프고 쨍하게 보이지 않도록 기본 색상을 더 편안하게 낮춘다. 사용자가 직접 `편안/따뜻/세이지/차분` 같은 색상 테마를 바꿔 보며 확인할 수 있어야 한다. 조리 화면은 기존 다크 모드를 유지한다.
15. 하단 탭바는 `Timers.png`의 아이폰 시계 앱처럼 하단에 떠 있는 둥근 캡슐 구조를 따르되, 순검정이 아니라 앱의 따뜻한 잉크톤에 맞춰 화면 안에 녹아들게 구성한다. 아이콘 위/라벨 아래 구조를 유지하고, 활성 탭은 내부의 더 밝은 둥근 캡슐과 오렌지 색상으로 표시한다. 가운데 `등록`도 별도 돌출 카드가 아니라 동일한 탭 규칙을 따른다.
16. 사용자가 실제 YouTube/Shorts/Instagram 링크를 붙여넣는 순간부터 그 링크가 활성 레시피가 되어야 한다. 상세 화면의 제목/썸네일/재료/단계와 cook3 조리 단계의 영상 구간 라벨이 모두 같은 링크 데이터에서 나와야 하며, 실제 재생을 단정하지 않고 원본 링크·구간 연결 상태를 정직하게 보여준다.

## Ralph 스토리 분해

### MVP-01 요구사항 문서화·PRD 등록
- 이 문서를 생성한다.
- `scripts/ralph/prd.json`에 `MVP-*` 스토리를 `passes:false`로 등록한다.
- 검증: 문서와 PRD가 같은 요구를 담고, `validate-prd.sh`가 통과한다.

### MVP-02 상세 CTA 투명 플로팅 보정
- 상세 화면의 `요리 시작하기`는 플로팅 버튼만 보이고, 하단 직사각형 배경이 콘텐츠를 가리지 않는다.
- 검증: `#detail` 스크린샷에서 CTA 주변 배경이 보이고 버튼 클릭 시 `#cook3`로 이동한다.

### MVP-03 조리 모드 종료·스와이프 페이싱
- cook3 하단에서 이전/다음 버튼을 제거한다.
- `요리 완성` 버튼을 추가한다.
- 단계 카드는 스와이프/휠/키보드로 이동 가능하고 힌트를 보여준다.
- 검증: 완료 버튼 클릭 시 `#complete`, 스와이프 시 단계 번호 변경.

### MVP-04 완료·팁·후기 액션 분리
- 완료 화면은 완성 결과를 먼저 보여준다.
- `팁 남기기`는 별도 화면으로 이동한다.
- `후기 남기기`는 기존 후기 작성 시트로 이동한다.
- MVP 밖 기능은 다음 버전 안내만 한다.
- 검증: 각 버튼이 하나의 액션만 수행한다.

### MVP-05 만든 요리 기록 카피 정합
- `최근 만든 요리`의 근거는 완료 일시, 메모/팁 여부, 다시 만들기 같은 재방문 동기로 표현한다.
- `성공`과 `다시 만들기`처럼 서로 다른 차원의 상태를 나란히 두지 않는다.
- 검증: `0회 터치`, 의미 불명 상태 문구가 남지 않는다.

### MVP-06 버튼별 클릭 매트릭스 검증
- 홈, 상세, 조리, 완료, 팁 화면의 주요 버튼을 CDP로 하나씩 클릭한다.
- 각 버튼은 화면 전환, 시트 열림, 토스트 중 하나를 만든다.
- 검증: 콘솔 에러 0, dead UI 0.

### MVP-07 외부 영상 소스 테스트 케이스 정리
- YouTube long/short, Instagram, 전문 요리사/일반 사용자 케이스를 샘플 데이터로 분리한다.
- 등록 시트에서 케이스를 선택해 내부 분석 상태가 바뀌도록 한다.
- 검증: 각 케이스가 플랫폼/제작자 유형/영상 길이/정직한 미리보기 상태를 보여준다.

### MVP-08 Figma·기술명세서 백로그 등록
- Figma MCP로 와이어프레임 제작.
- Notion 또는 Excel에 기술명세서 제작.
- 지금은 실행하지 않고 후속 백로그로만 둔다.
- 검증: TODO 문서 또는 PRD 백로그에 남는다.

### MVP-09 요리북 세부 페이지 분리
- 홈에서 최근 만든 요리 섹션을 제거한다.
- 하단 `요리북` 탭은 별도 `cookbook` 페이지로 이동한다.
- 요리북에는 최근 조리, 저장한 레시피, 기록 기반 추천, 기록 메모를 보여준다.
- 검증: 홈 스크린샷에 최근 만든 요리가 없고, `#cookbook` 스크린샷에 기록/저장/추천이 보인다.

### MVP-10 조리 과정 블랙·화이트 컬러 변형
- `cook3` 조리 과정만 블랙 배경과 화이트 텍스트 중심으로 유지한다.
- 기본안은 화이트 액센트로 두고, 보조안은 허브/스틸/크림 액센트로 비교한다.
- 진행바, 단계 카드, 타이머, 핸즈프리 버튼의 액센트만 바뀌고 홈/상세/요리북 전역 색상은 바꾸지 않는다.
- 검증: `#cook3`에서 4개 색상 칩 클릭 시 `data-cook-tone`이 바뀌고 콘솔 에러가 없어야 한다.

### MVP-11 라이트 일반 화면·3탭 내비게이션
- 홈/상세/요리북/완료/팁/등록 시트는 라이트 화면으로 전환한다.
- `cook3`, `cook`, `cook2` 조리 화면은 다크 모드로 유지한다.
- 하단 내비게이션은 왼쪽 `홈`, 가운데 `등록`, 오른쪽 `요리북`만 둔다.
- 설정은 홈 상단 계정 아이콘에서 진입한다.
- 검증: 홈과 요리북 하단 탭이 3개이고 `settings` 탭이 없어야 하며, `#cook3` 배경은 다크여야 한다.

### MVP-12 레시피 등록 링크 단일 입력
- 등록 시트에는 영상 링크 입력만 둔다.
- 사용자가 출처, 플랫폼, 영상 길이, 제작자 유형을 직접 고르는 UI를 제거한다.
- 링크 입력값을 기준으로 앱이 내부적으로 YouTube/Shorts/Instagram 계열을 자동 감지해 미리보기를 바꾼다.
- 검증: 등록 시트에 `.source-case` 선택 버튼이 0개이고, 링크 입력 후 미리보기만 갱신되어야 한다.

### MVP-13 눈 편한 색상 테마 선택
- 기본 라이트 팔레트의 밝기와 채도를 낮춘다.
- 홈 상단 계정 버튼에서 색상 설정 시트를 연다.
- `편안`, `따뜻`, `세이지`, `차분` 테마를 직접 눌러 즉시 비교할 수 있게 한다.
- 선택 테마는 로컬에 저장되어 다음 실행에도 유지된다.
- 검증: 4개 테마 버튼 클릭 시 전역 배경/액센트가 바뀌고, `cook3` 조리 화면은 다크 배경을 유지해야 한다.

### MVP-14 iOS 시계형 하단 탭바
- 하단 탭바를 화면 하단에 떠 있는 iOS 시계 앱 `Timers.png` 계열의 둥근 캡슐로 구성한다.
- 색상은 순검정보다 앱의 따뜻한 잉크톤을 사용해 라이트 화면과 자연스럽게 연결한다.
- 탭은 `홈 / 등록 / 요리북` 3개를 유지한다.
- 각 탭은 아이콘 위, 라벨 아래 구조를 유지한다.
- 선택 탭은 내부의 더 밝은 둥근 캡슐 배경과 오렌지 아이콘/라벨로 강조한다.
- 가운데 `등록` 탭은 별도 플로팅/그라데이션 버튼처럼 보이지 않고 다른 탭과 같은 규칙을 따른다.
- 검증: 홈/요리북 스크린샷에서 하단 탭 3개, 플로팅 캡슐 탭바, 앱 팔레트와 맞는 따뜻한 어두운 배경, 비활성 흰색 계열, 활성 오렌지+내부 캡슐, 중앙 등록 탭의 별도 카드 스타일 제거가 확인되어야 한다.

### MVP-15 실제 영상 링크 기반 단계별 구간 바인딩
- 레시피 등록 시트에 붙여넣은 링크를 `activeRecipe`로 만든다.
- YouTube/Shorts 링크는 실제 video id를 파싱해 `i.ytimg.com` 썸네일과 단계별 원본 구간 URL을 구성한다.
- Instagram 링크는 대체 썸네일과 원본 확인 상태를 사용하고, 직접 재생을 단정하지 않는다.
- 상세 화면의 제목, 썸네일, 플랫폼 배지, 재료, 단계 목록, 후기/저장 대상은 활성 레시피 기준으로 바뀐다.
- cook3 단계 카드와 영상 영역은 현재 단계의 시간 구간, 플랫폼, 원본 구간 URL을 함께 바꾼다.
- 검증: Shorts 링크 입력 후 상세가 4단계/재료 6개/YouTube Shorts로 바뀌고, cook3에서 1단계→2단계 이동 시 영상 구간 라벨과 `data-url`이 함께 변경되어야 한다.
