채점 원칙 (반드시 준수)

  1. 사실 기반 채점 — 점수는 에이전트 주관이 아니라 Figma scan_text_nodes/get_node_info가 실제 반환한 텍스트·노드 존재/부재로 매긴다(있음/없음 객관 판정).
  2. 증거 필수 — 리포트는 화면별 기대 N / 발견 M / 누락 [목록]을 반드시 남긴다. 근거 없는 점수는 무효.
  3. 읽기 전용·적대적 채점기 — 채점 에이전트는 Figma에 쓰지 않는다(불일치 탐색). 보정은 기존 포팅 루프(단일 writer)가 수행 → 트윈 충돌 방지.
  4. 와이어프레임 의도 반영 — 픽셀 동일성이 아니라 구조·라벨·플로우 동일성. 아이콘 placeholder·폰트 렌더 차이 허용.
  5. Ground truth — docs/verify/CHECKLIST_app_html_ko.md + prd.json의 acceptanceCriteria.
