# Cookflow 와이어프레임 → Figma 이식 스토리라인

> 목적: `app.html`(영상 레시피 핸즈프리 요리 어시스턴트 "Cookflow", 페르소나 민지)의 각 화면을 **Figma 캔버스에 실제로 그려 넣는다**. HTML은 수정하지 않는다.
> 방식: `scripts/ralph-figma/` ralph 루프로 **화면 1개 = 스토리 1개**씩 자율 구현 → green-gate 통과 → 한글 커밋.
> 원본 목표 이미지: `scripts/ralph-figma/source-screens/<view>.png` (헤드리스 Chrome 438×892 캡처).

## 0. 전제 (쓰기형 Figma MCP)
- 그리기는 **쓰기 가능한 플러그인 브리지형 Figma MCP**가 필요하다(Framelink/REST/Dev Mode MCP는 읽기 전용이라 불가).
- 사용자 셋업(플러그인 설치·대상 파일 열기·브리지 실행·채널 조인·세션 재시작) 완료 후에만 실제 그리기가 가능하다. 셋업 절차는 `scripts/ralph-figma/FIGMA_SETUP_ko.md` 참조.
- MCP 툴은 세션 시작 시 바인딩되므로 `.mcp.json`을 추가한 뒤 **세션 재시작**이 필요하다.

## 1. Figma 문서 구조 (목표)
- 페이지 1개: `Cookflow Wireframes`.
- 화면마다 최상위 프레임 `cook/<view>`, 모바일 크기 **390×844**, 세로 auto-layout.
- 프레임을 플로우 순서대로 가로 배치(간격 120px). 공통 요소(상태바·하단탭바)는 컴포넌트/재사용 프레임으로.

## 2. Green-gate (스토리 통과 기준, 모든 화면 공통)
1. **G0 연결** — 채널 조인 + `get_document_info` 성공.
2. **G1 그리기** — `cook/<view>` 프레임 생성, 원본 영역(상태바→헤더→콘텐츠→하단) 재현.
3. **G2 read-back** — `get_node_info`로 자식 노드 ≥ 6, 그 화면의 핵심 한글 라벨 텍스트 노드 ≥ 1 존재.
4. **G3 export** — `export_node_as_image` PNG(scale≥2)를 `scripts/ralph-figma/exports/<view>.png`로 저장, 원본과 영역 대응 대조.
5. **G4 정직/토큰** — 영어 누수 0, 브랜드/다크 토큰 사용, 빈·중복·깨진 프레임 없음.

## 4. 스토리 (화면별 구체화)

### F0 — 연결 스파이크 (home 1화면 end-to-end)
- **목적**: 파이프라인(연결→그리기→read-back→export) 증명. 이게 통과해야 나머지를 신뢰한다.
- **할 일**: 채널 조인 확인 → `cook/home` 프레임에 상태바·로고·헤드라인만 최소 그리기 → read-back → export.
- **통과**: G0~G3 성공, `exports/home_spike.png` 생성.

### F1-01 — home (홈)
- **영역**: ① 상태바(9:41·신호/와이파이/배터리) ② 헤더: 로고 `Cookflow` + "영상 레시피 요리 도우미", 우측 알림·프로필 아이콘 ③ 헤드라인 "오늘 뭐 만들까요?" ④ **영상 링크로 바로 요리하기** 오렌지 카드(+ "재료, 단계, 구간을 먼저 확인하고 시작") ⑤ 검색바 "먹고 싶은 요리를 검색해보세요" ⑥ 카테고리 원형칩 5개(셰프·급상승·한식·분식·일식) ⑦ 섹션 "당신만을 위한 추천" + "전체 >" ⑧ 추천 카드 2개(썸네일 + 추천사유 캡션 + 제목 + "저자·인분·시간·기록 기반") ⑨ 하단탭바(홈·등록·요리북, 가운데 등록 강조).
- **핵심 라벨(G2)**: "오늘 뭐 만들까요?", "영상 링크로 바로 요리하기".

### F1-02 — detail (레시피 상세)
- **영역**: ① 상태바 ② 헤더(뒤로·"레시피"·공유) ③ 영상 히어로(썸네일 + 재생아이콘 + **"정적 미리보기"** 배지 + "YouTube" 배지 — 거짓 재생 표기 금지) ④ 제목 "백종원이 알려주는 칼칼한 순두부찌개" + 북마크·저장수(342) ⑤ ⭐4.7 · 후기 3개 ⑥ 설명 문단 ⑦ **먼저 만든 사람의 팁** 카드 ⑧ 메타칩(5단계 추출·재료 9개·타임스탬프 연동) ⑨ 하단 플로팅 CTA **"요리 시작하기"**(그린/오렌지, 투명 플로팅).
- **핵심 라벨(G2)**: "요리 시작하기", "정적 미리보기".

### F1-03 — loading (영상 분석 로딩)
- **영역**: 중앙 로딩 인디케이터 + "영상을 분석하고 있어요" 류 상태 카피 + 단계 진행 힌트(재료/단계 추출 중). 정직한 진행 표기, 가짜 완료 금지.
- **핵심 라벨(G2)**: 분석 상태 카피 1개.

### F1-04 — cook3 (조리 모드, 주력 ●●)
- **영역(다크)**: ① 상태바(다크) ② 헤더(뒤로·"조리 모드"·도움말?) ③ 영상 영역 + **"정적 미리보기 · 실제 재생은 실기기에서 확인"** 오버레이(거짓 재생중 금지) ④ 단계 진행 도트(5칸, 1칸 활성) ⑤ 톤 선택칩(화이트·허브·스틸·크림) ⑥ "단계 카드를 위아래로 스와이프" 힌트 ⑦ 현재 단계 카드(다크): "1단계. 육수 준비하기 · 1/5" + 번호 리스트 3줄 ⑧ 하단 컨트롤 4버튼: 타이머 / 핸즈프리 꺼짐 / 재료 / **요리 완성**(그린 원형 체크).
- **핵심 라벨(G2)**: "요리 완성", "1단계. 육수 준비하기".
- **주의**: 이전/다음 버튼 없음(스와이프 페이싱). 음성패널은 시뮬레이션 표기.

### F1-05 — complete (완료)
- **영역**: 완성 결과 강조(체크/사진) + "완성!" 류 헤드라인 + 요약(요리명·시간) + 액션(팁 남기기 → tipWrite / 홈으로 / 다시 요리하기). 무의미 지표(0회 터치 등) 없음.
- **핵심 라벨(G2)**: "완성", "팁 남기기".

### F1-06 — tipWrite (팁 남기기)
- **영역**: 헤더(뒤로·"팁 남기기") + 별점 선택 + 텍스트 입력 + 사진 첨부 + 등록 CTA. 등록 후 complete로 복귀.
- **핵심 라벨(G2)**: "팁 남기기".

### F1-07 — cookbook (요리북)
- **영역**: 헤더 "요리북" + 섹션: 최근 조리 / 저장한 레시피 / 기록 기반 추천 / 기록 메모. 최근 만든 요리와 저장 레시피 분리. 하단탭바.
- **핵심 라벨(G2)**: "요리북".

### F1-08 — reviews (후기 전체)
- **영역**: 헤더(뒤로·"후기") + 평점 요약(평균·개수) + 후기 리스트(작성자·별점·본문·사진). 하단 작성 CTA.
- **핵심 라벨(G2)**: "후기".

### F1-09 — sheet (영상 링크 등록 바텀시트)
- **영역**: 스크림 위 바텀시트 + 핸들 + "영상 링크 등록" 제목 + 링크 입력 + 플랫폼 배지(YouTube/Instagram) + 분석 시작 CTA. 정직한 처리 상태.
- **핵심 라벨(G2)**: "등록" 또는 "영상 링크".

### F1-10 — cook (조리 모드 구, 레거시)
- **영역(다크)**: 타이머/재료/음성 가이드 진입점이 있는 구버전 조리 화면. 레거시 표기.
- **핵심 라벨(G2)**: "조리".

### F1-11 — cook2 (핸즈프리 구, 레거시)
- **영역(다크)**: 정적 핸즈프리 스냅샷. 레거시 표기.
- **핵심 라벨(G2)**: 핸즈프리 관련 라벨 1개.

### F2-01 — 공통 컴포넌트화 + 플로우 정리
- **목적**: 상태바·하단탭바를 컴포넌트로 승격해 화면 간 재사용, 프레임을 플로우 순서로 정렬(home→sheet→loading→detail→cook3→complete→tipWrite, 보조: cookbook/reviews).
- **통과**: 컴포넌트 인스턴스가 최소 2화면에서 재사용됨을 read-back으로 확인.

## 5. 실행 순서
1. (사용자) `scripts/ralph-figma/FIGMA_SETUP_ko.md`대로 Figma 브리지 연결 + 세션 재시작.
2. **F0 스파이크**를 수동/1-iteration으로 검증(`./scripts/ralph-figma/ralph.sh --tool claude 1`).
3. 통과 시 자율 루프: `./scripts/ralph-figma/supervise.sh 3 3` (미완료 0까지).
4. 산출: `docs/progress/FIGMA_REPORT_ko.md`(스토리별) · `exports/<view>.png` · 완주 시 최종 리포트.
